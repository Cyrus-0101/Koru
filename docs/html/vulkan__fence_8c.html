Detailed Description

Implementation of Vulkan fence creation, waiting, and destruction.

Fences are synchronization primitives that allow the CPU to wait for the GPU to finish tasks. They are central to managing multiple frames-in-flight safely.

Creates a new Vulkan fence.

Fences are used to synchronize the CPU with GPU operations. A fence starts in either:

  • Unsignaled state (default)
  • Signaled state (create_signaled == True)
Parameters
contextA pointer to the active Vulkan context.
create_signaledIf True, the fence will be created in a signaled state.
out_fenceA pointer to the vulkan_fence struct to populate.

Waits for the given fence to be signaled by the GPU.

If the fence is already signaled, returns immediately. Otherwise, waits up to timeout_ns nanoseconds for it to become signaled.

Parameters
contextA pointer to the active Vulkan context.
fenceA pointer to the vulkan_fence to wait for.
timeout_nsMaximum time to wait in nanoseconds.
Returns
True if the fence was signaled within the timeout; False otherwise.
