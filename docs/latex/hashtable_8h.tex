\doxysubsection{Detailed Description}
Represents a simple hashtable. Members of this structure should not be modified outside the functions associated with it. 

For non-\/pointer types, table retains a copy of the value. For pointer types, make sure to use the \+\_\+ptr setter and getter. Table does not take ownership of pointers or associated memory allocations, and should be managed externally.

Usage\+:
\begin{DoxyItemize}
\item Create a hashtable with {\ttfamily \doxylink{hashtable_8h_a9ae8fa47b0e55edfb2b93cf02bb72d7e}{hashtable\+\_\+create()}}, providing element size, count, memory block, and type info.
\item Use {\ttfamily \doxylink{hashtable_8h_aa34d99cd3721459425c8fdb5b862eecf}{hashtable\+\_\+set()}} and {\ttfamily \doxylink{hashtable_8h_a47e12a32fbf7a084ee50e4e8487e633e}{hashtable\+\_\+get()}} for non-\/pointer types.
\item Use {\ttfamily \doxylink{hashtable_8h_a0f8dffa45ec24f5df3010d7331991832}{hashtable\+\_\+set\+\_\+ptr()}} and {\ttfamily \doxylink{hashtable_8h_ac8ad4fa00bd9c27020f80fc7915c1027}{hashtable\+\_\+get\+\_\+ptr()}} for pointer types.
\item Destroy the hashtable with {\ttfamily \doxylink{hashtable_8h_ae3e85997276b1152d2c62fa2694e1c14}{hashtable\+\_\+destroy()}}. Note\+:
\item The hashtable does not handle collisions; ensure unique keys or increase element count. 
\end{DoxyItemize}
